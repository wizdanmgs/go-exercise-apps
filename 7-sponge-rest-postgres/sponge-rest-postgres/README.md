# Sponge Rest Progress

A production-oriented REST API built with Sponge Framework and PostgreSQL using migrations, repository pattern, connection pooling, and Basic Authentication.

## Technology Stack

- Programming Language: Go
- Web Framework: Gin
- Configuration Management: Viper
- Logging: Zap
- ORM: Gorm
- Database: PostgreSQL
- Caching: Go-Redis
- Monitoring: Prometheus + Grafana
- Tracing: OpenTracing + Jaeger
- Other: ...

## Directory Structure

```text.

├─ cmd                          # Application entry directory
│   └─ sponge_rest_postgres                     # Service name
│       ├─ initial              # Initialization logic (e.g., configuration loading, service initialization)
│       └─ main.go              # Main program entry file
├─ configs                      # Configuration directory (YAML format templates)
├─ deployments                  # Deployment scripts (binary, Docker, K8S deploy)
├─ docs                          # Project documentation (API docs, design docs, etc.)
├─ internal                      # Internal implementation code (not publicly exposed)
│   ├─ cache                      # Caching implementation (Redis or local memory cache wrapper)
│   ├─ config                    # Configuration parsing and struct definitions
│   ├─ dao                      # Data Access Layer (Database Access Object)
│   ├─ ecode                    # Error code definitions
│   ├─ handler                  # Business logic processing layer (similar to Controller)
│   ├─ model                    # Data model/entity definitions
│   ├─ routers                  # Routing definitions and middleware
│   ├─ server                   # Service startup
│   └─ types                    # Request/Response Structure Definitions
├─ scripts                      # Utility Scripts (Code Generation, Build, Run, Deployment, etc.)
├─ go.mod                       # Go Module Definition File (Declares Dependencies)
├─ go.sum                       # Go Module Verification File (Autogenerated)
├─ Makefile                     # Project Build Automation Script
└─ README.md
```

The code adopts a layered architecture with the complete call chain as follows:

`cmd/sponge_rest_postgres/main.go` → `internal/server/http.go` → `internal/routers/router.go` → `internal/handler` → `internal/dao` → `internal/model`

The handler layer primarily handles API processing. For more complex business logic, it is recommended to add an additional business logic layer between the handler and DAO layers (e.g., `service`, `logic`, or `biz`—define as needed).

## Quick Start

### 1. Generate OpenAPI Documentation

```bash
make docs
```

Note: Execute this command only when adding or modifying APIs. Do not repeat when APIs remain unchanged.

### 2. Compile and Run

```bash
make run
```

### 3. Test APIs

Access [http://localhost:8080/swagger/index.html](http://localhost:8080/swagger/index.html) in your browser to test the HTTP API.
